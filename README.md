# Webapp Template

A full-stack webapp template using a pnpm monorepo.

## Stack

- **Frontend** (`packages/web`): React 19, Vite, Tailwind CSS v4, DaisyUI v5, AsterUI, TanStack Query, React Router, Jotai
- **Backend** (`packages/api`): Hono, PetraDB (in-memory SQL), Drizzle ORM, cookie-based JWT sessions
- **Monorepo**: pnpm workspaces with Hono RPC for end-to-end type safety

## Getting Started

```bash
pnpm install
pnpm dev
```

The frontend runs on `http://localhost:5173` and proxies `/api` to the backend on port 3000.

## Structure

```
packages/
  api/
    drizzle.config.ts  # Drizzle Kit config (points to schema + migrations output)
    src/
      db/
        schema.ts      # Drizzle ORM table definitions + relations
        migrations/    # Generated by drizzle-kit, applied at startup
        seed.ts        # Runs migrations then inserts demo data
        index.ts       # DB connection (Session + drizzle())
      routes/          # Hono route handlers
      app.ts           # Hono app (exports AppType for RPC)
      server.ts        # Entry point
      session.ts       # Cookie-based JWT session (@edadma/hono-session)
  web/
    src/
      api.ts           # Hono RPC client (type-safe via AppType)
      useAuth.ts       # Auth hook (TanStack Query)
      App.tsx           # Routes with ProtectedRoute/GuestRoute
      LoginPage.tsx     # AsterUI login form
      HomePage.tsx      # Authenticated home page
      main.tsx          # Entry point (QueryClient, ThemeProvider, BrowserRouter)
      index.css         # Tailwind + DaisyUI + AsterUI config
```

## Database & Migrations

Schema lives in `packages/api/src/db/schema.ts` using Drizzle ORM's `pgTable` definitions with relations.

**Workflow after changing the schema:**

```bash
pnpm --filter @my-app/api db:generate
```

This runs `drizzle-kit generate` which reads the schema and produces SQL migration files in `src/db/migrations/`. Migrations are applied automatically at server startup via `migrate()` from `@petradb/drizzle`.

**Use relational queries** (`db.query.*`) with `findFirst`/`findMany` and `with:` for loading related data instead of manual joins:

```ts
const user = await db.query.users.findFirst({
  where: eq(schema.users.id, userId),
  columns: { id: true, username: true, fullName: true },
  with: { workspace: true },
})
```

## Auth

- Sessions use **httpOnly cookies** with JWT tokens via `@edadma/hono-session` (sameSite: Strict, secure in production)
- `session.login(c, payload)` sets the cookie, `session.logout(c)` deletes it
- `session.authMiddleware` verifies the token and sets `c.get('session')` with the payload
- Frontend `useAuth()` hook calls `GET /api/auth/me` via TanStack Query, caches for 5 minutes
- `ProtectedRoute` redirects to `/login` if unauthenticated; `GuestRoute` redirects to `/` if already logged in

## AsterUI Setup

CSS order in `index.css` matters:

```css
@import "tailwindcss";
@plugin "daisyui" { ... }          /* daisyui FIRST */
@source "../node_modules/asterui";  /* asterui source AFTER */
```

The app must be wrapped in `<ThemeProvider>` from asterui (see `main.tsx`).

AsterUI templates for login/signup forms are at `~/dev/asterui/packages/docs/src/templates/`.

## API Patterns

- API calls from the frontend go through **TanStack Query** (`useQuery`/`useMutation`) + the **Hono RPC client** (`client.api.auth.login.$post(...)`)
- Route handlers are typed Hono apps that get composed in `app.ts` — the exported `AppType` gives the frontend full type safety
- Use `db.query.*` relational queries where possible instead of raw `db.select().from()`
